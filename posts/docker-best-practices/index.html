Docker 容器化部署最佳实践.

为什么选择 Docker?

Docker 已经成为现代应用开发的标准工具,它解决了"在我机器上能运行"的经典问题:

  • 环境一致性: 开发、测试、生产环境完全一致
  • 快速部署: 秒级启动,弹性伸缩
  • 资源隔离: 容器间相互独立,互不影响
  • 版本管理: 镜像版本控制,轻松回滚

Dockerfile 编写技巧

基础镜像选择

选择合适的基礎镜像至关重要:

# ❌ 不推荐:使用 latest 标签,不可控
FROM node:latest

# ✅ 推荐:指定具体版本,使用 Alpine 减小体积
FROM node:18-alpine

# ✅ 更佳:使用 LTS 版本的精简版
FROM node:18.19-alpine3.19

常用基础镜像大小对比:

  • node:18: ~900MB
  • node:18-slim: ~200MB
  • node:18-alpine: ~120MB

多阶段构建

多阶段构建可以显著减小最终镜像体积:

# 第一阶段:构建
FROM node:18-alpine AS builder

WORKDIR /app

# 先复制依赖文件,利用缓存
COPY package*.json ./
RUN npm ci --only=production

# 复制源代码并构建
COPY . .
RUN npm run build

# 第二阶段:运行
FROM node:18-alpine AS production

WORKDIR /app

# 只复制必要的文件
COPY --from=builder /app/package*.json ./
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/dist ./dist

# 使用非 root 用户运行
RUN addgroup -g 1001 -S appgroup && \
    adduser -S appuser -u 1001 -G appgroup
USER appuser

EXPOSE 3000

CMD ["node", "dist/index.js"]

优化层缓存

合理利用 Docker 的层缓存机制:

FROM node:18-alpine

WORKDIR /app

# ✅ 先复制依赖配置文件
COPY package*.json ./

# 安装依赖(这一步会被缓存,除非 package.json 改变)
RUN npm ci --only=production

# 再复制源代码(经常变化的部分放在后面)
COPY . .

# 构建应用
RUN npm run build

EXPOSE 3000
CMD ["node", "dist/index.js"]

Docker Compose 实战

完整的应用栈配置

version: '3.8'

services:
  # Web 应用
  app:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - DATABASE_URL=postgresql://user:password@db:5432/mydb
      - REDIS_URL=redis://redis:6379
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_started
    restart: unless-stopped
    networks:
      - app-network
    volumes:
      - app-data:/app/data
    deploy:
      resources:
        limits:
          cpus: '1'
          memory: 512M

  # PostgreSQL 数据库
  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: mydb
      POSTGRES_USER: user
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    volumes:
      - postgres-data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U user -d mydb"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - app-network
    restart: unless-stopped

  # Redis 缓存
  redis:
    image: redis:7-alpine
    command: redis-server --appendonly yes
    volumes:
      - redis-data:/data
    networks:
      - app-network
    restart: unless-stopped

  # Nginx 反向代理
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - ./ssl:/etc/nginx/ssl:ro
    depends_on:
      - app
    networks:
      - app-network
    restart: unless-stopped

volumes:
  postgres-data:
  redis-data:
  app-data:

networks:
  app-network:
    driver: bridge

安全最佳实践

1. 最小权限原则

# 创建专用用户和组
RUN addgroup -g 1001 -S appgroup && \
    adduser -S appuser -u 1001 -G appgroup

# 设置文件所有权
COPY --chown=appuser:appgroup . .

# 切换到非 root 用户
USER appuser

2. 敏感信息管理

# ❌ 错误:硬编码敏感信息
ENV DATABASE_PASSWORD=mysecretpassword

# ✅ 正确:使用环境变量或密钥管理
ARG DB_PASSWORD
ENV DATABASE_PASSWORD=${DB_PASSWORD}

构建时传入:

docker build --build-arg DB_PASSWORD=${SECRET_PASSWORD} -t myapp .

3. 镜像扫描

使用工具扫描镜像中的安全漏洞:

# 安装 Trivy
brew install trivy

# 扫描镜像
trivy image myapp:latest

# 在 CI/CD 中集成
trivy image --exit-code 1 --severity HIGH,CRITICAL myapp:latest

性能优化技巧

1. 使用 .dockerignore

node_modules
npm-debug.log
.git
.gitignore
README.md
.env
.dockerignore
Dockerfile
docker-compose.yml
.vscode

2. 并行构建优化

# 利用 BuildKit 并行构建
# syntax=docker/dockerfile:1

FROM node:18-alpine

WORKDIR /app

# 并行安装依赖
RUN --mount=type=cache,target=/root/.npm \
    npm ci --only=production

COPY . .
RUN npm run build

启用 BuildKit:

export DOCKER_BUILDKIT=1
docker build -t myapp .

3. 健康检查

HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD wget --no-verbose --tries=1 --spider http://localhost:3000/health || exit 1

监控和日志

日志管理

services:
  app:
    # ... 其他配置
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

查看日志:

# 查看实时日志
docker logs -f myapp

# 查看最近 100 行
docker logs --tail 100 myapp

# 查看特定时间范围
docker logs --since 2024-01-20T10:00:00 myapp

CI/CD 集成示例

GitHub Actions 工作流:

name: Docker Build and Deploy

on:
  push:
    branches: [ main ]

jobs:
  build-and-push:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Login to Docker Hub
      uses: docker/login-action@v2
      with:
        username: ${{ secrets.DOCKER_USERNAME }}
        password: ${{ secrets.DOCKER_PASSWORD }}
    
    - name: Build and push
      uses: docker/build-push-action@v4
      with:
        context: .
        push: true
        tags: myuser/myapp:latest,myuser/myapp:${{ github.sha }}
        cache-from: type=registry,ref=myuser/myapp:buildcache
        cache-to: type=registry,ref=myuser/myapp:buildcache,mode=max
    
    - name: Run security scan
      uses: aquasecurity/trivy-action@master
      with:
        image-ref: myuser/myapp:${{ github.sha }}
        format: 'table'
        exit-code: '1'
        severity: 'CRITICAL,HIGH'

常见问题排查

1. 容器无法启动

# 查看容器状态
docker ps -a

# 查看详细日志
docker logs <container_id>

# 进入容器调试
docker exec -it <container_id> sh

2. 网络连接问题

# 查看网络
docker network ls

# 检查容器网络
docker inspect <container_id> | grep IPAddress

# 测试连通性
docker exec <container_id> ping db

3. 磁盘空间清理

# 查看磁盘使用情况
docker system df

# 清理未使用的资源
docker system prune -a

# 清理悬空镜像
docker image prune

总结

Docker 容器化部署是现代应用开发的重要技能。通过掌握多阶段构建、镜像优化、安全配置等技术,我们可以构建出高效、安全的容器化应用。记住以下关键点:

  1. 选择合适的基礎镜像,优先使用 Alpine 版本
  2. 使用多阶段构建减小镜像体积
  3. 遵循最小权限原则,不使用 root 用户
  4. 合理管理敏感信息,避免硬编码
  5. 利用层缓存机制加速构建过程
  6. 定期扫描安全漏洞,保持镜像安全

持续学习和实践,你将能够熟练运用 Docker 提升开发和部署效率!
