图片使用指南

图片存放位置

将图片文件放置在以下目录:

static/images/

在文章中使用图片

方法 1: Front Matter 中设置封面图

---
title: 文章标题
image: images/cover.jpg
imageAltAttribute: 图片描述文字
---

方法 2: 在文章内容中插入图片

![图片描述](/images/example.jpg)

<!-- 或者使用 HTML -->
<img src="/images/example.jpg" alt="图片描述" />

推荐的图片格式和尺寸

封面图片

  • 格式: JPEG 或 WebP
  • 尺寸: 1200x630 像素 (1.91:1 比例)
  • 文件大小: 建议小于 200KB

文章内图片

  • 格式: JPEG、PNG 或 WebP
  • 宽度: 800-1200 像素
  • 文件大小: 建议小于 100KB

作者头像

  • 文件名: author.png 或 author.jpg
  • 尺寸: 400x400 像素 (正方形)
  • 格式: PNG (透明背景) 或 JPEG

社交图标

放置在 static/images/social/ 目录:

  • github.svg
  • twitter.svg
  • linkedin.svg

图片优化建议

1. 压缩图片

使用工具压缩图片以减小文件大小:

  • TinyPNG - PNG/JPEG 压缩
  • Squoosh - 在线图片优化工具
  • ImageOptim - Mac 图片优化应用

2. 使用现代格式

考虑使用 WebP 格式以获得更好的压缩率:

# 使用 cwebp 转换
cwebp -q 80 image.png -o image.webp

3. 响应式图片

对于重要图片,可以考虑提供多种尺寸:

<picture>
  <source srcset="/images/photo-small.jpg" media="(max-width: 600px)">
  <source srcset="/images/photo-medium.jpg" media="(max-width: 1200px)">
  <img src="/images/photo-large.jpg" alt="描述">
</picture>

故障排除

图片不显示

  1. 检查文件路径是否正确
  2. 确认图片文件存在于 static/images/ 目录
  3. 检查文件大小写是否匹配
  4. 清除浏览器缓存后重试

图片加载缓慢

  1. 压缩图片文件大小
  2. 考虑使用懒加载
  3. 使用 CDN 加速
  4. 转换为 WebP 格式

图片变形

确保在 CSS 中设置正确的宽高比:

img {
  max-width: 100%;
  height: auto;
}
